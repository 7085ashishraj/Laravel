# Unit I: Getting Started with MVC Laravel Framework

## What is MVC Framework?
MVC stands for Model-View-Controller. It is a software design pattern that separates application logic into three interconnected components:
- **Model:** Manages data, logic, and rules of the application (e.g., interacting with the database).
- **View:** Represents the UI and displays the data to the user.
- **Controller:** Accepts input from the user (via the View), processes it (using the Model), and returns the output.

**Example:**
Imagine a restaurant:
- The **View** is the menu and the table where the customer sits.
- The **Controller** is the waiter who takes your order and brings it to the kitchen.
- The **Model** is the chef who prepares the food based on the ingredients (database).

## Overview of Laravel Framework and its Features
Laravel is a free, open-source PHP web framework created by Taylor Otwell, intended for the development of web applications following the MVC architectural pattern. 

**Key Features:**
1. **Eloquent ORM:** An advanced Object-Relational Mapper to interact with databases beautifully.
2. **Blade Templating Engine:** A powerful, lightweight templating engine for views.
3. **Artisan CLI:** A command-line interface packed with helpful commands.
4. **Routing System:** Easy and expressive routing.
5. **Middleware:** Filtering HTTP requests.
6. **Security:** Built-in protection against SQL injection, CSRF, and XSS.
